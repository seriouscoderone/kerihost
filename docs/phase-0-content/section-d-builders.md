# Section D — "What This Enables for Builders"

**Type:** Essay / guide

**Priority:** ⚪ Ongoing/Flexible (Phase 1)

---

## Purpose

Developers enter as *enablers*, not heroes. Position developers as "infrastructure gardeners, not platform owners."

## Framing

- "If you want to build tools for communities like this…"
- "If you want to offer services without becoming a gatekeeper…"
- "If you want to help communities coordinate without owning them…"

## Outline

### The Builder's Role
- Service provider, not platform owner
- Maintainer, not controller
- Gardener, not architect

### What You Can Build
- Witness services
- Credential issuance tools
- Verification interfaces
- Integration bridges
- Community-specific UX

### What You Should Avoid Building
- Central registries
- Universal reputation
- Required dependencies
- Lock-in mechanisms

### Getting Started
- Reference implementations
- Pattern library
- Community resources
