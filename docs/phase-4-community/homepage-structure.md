# KERI.host Homepage Structure

**Type:** Information architecture / wireframe document

**Priority:** ⚪ Ongoing/Flexible (Phase 0–1)

---

## Purpose

Design the public face of KERI.host as a narrative hub.

## Suggested Structure

### Hero Section
**"Building Communities Without Gatekeepers"**

- One-sentence value proposition
- Visual: Community, not platform
- CTA: Learn more / Read the manifesto

### Section A Teasers (The Problems)
- "Why communities struggle at scale"
- "Why platforms start helpful and end extractive"
- "Why trust doesn't scale, but accountability can"

Link to full essays

### Section C Story Previews (Ecosystem Stories)
- Homeschool network card
- Trades cooperative card
- City permit office card

"See yourself in these stories"

### "What We Don't Do" Callout
- Prominent anti-pattern list
- Builds trust through explicit limits
- Links to full document

### Builder Entry Point (Section D)
- "For developers who want to enable, not own"
- Technical getting started
- Reference implementations

### "What KERI.host Actually Is" (Section E)
- Humble, clear positioning
- Reference library, not platform
- Success = you don't need us

### Footer
- Documentation links
- GitHub repository
- Community channels
- License information

## Design Principles

1. **Narrative first** — Stories before specifications
2. **Trust through limits** — Emphasize what we don't do
3. **Multiple entry points** — Different audiences, different paths
4. **No hype** — Grounded, honest, quietly radical
